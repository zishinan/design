## 《设计模式入门与精通》源码
### OO基础
- 抽象
- 封装
- 多态
- 继承

### OO原则
- 封装变化
- 多用组合，少用继承
- 针对接口编程，不针对实现编程
- 为交互对象之间松耦合设计而努力
- 类应该对扩展开放，对修改关闭
- 依赖抽象，不依赖具体类
- 最少知识原则：只和你的密友交谈

### OO模式
- 命令模式
- 代理模式
- 装饰者模式
- 适配器模式：将一个类的接口，转换成客户期望的另一个接口。适配器让原本不兼容的类可以协作无间。
- 外观模式：提供一个统一的接口，用来访问子系统中的一群接口。外观定义了一个高层接口，让子系统更容易使用。
